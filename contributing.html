Contributing: Shaping the next NEBA

Bugs, Features and other issues

NEBA welcomes contribution of any form at any time. If you'd like to a report an issue or request a feature or change, just created an issue - a responsible committer will get to it.

Contributing source code

If you would like to contribute code, we're looking forward to your pull request. Note: The primary development branch is "develop" (we use gitflow), so you might want to develop against that.

Before we can integrate your change, we need you to sign the individual contributor license agreement and the corporate license agreement, if you contribute as a member of an organization. This is crucial to keep our stuff IP-clean - thank you!

Becoming a committer

If you have knowledge of both Spring and Sling and would like to join, you are more than welcome as a committer. Usually, you start of by making a couple of pull requests to show your expertise - than, the existing committers will vote on the decision.
