NEBA Release 3.2.0

Aug262014

Release 3.2.0

  • Posted by {% if page.twitterAuthor%}{% endif %}Olaf Otto{% if page.twitterAuthor %}{% endif %}

This release hardenes the NEBA taglib in AEM 5.6.x against issues with fragment bundles in Felix and features a delivery package that can automatically be deployed using the content-package-maven-plugin.

Change requests

#4 Remove classifier (-aem) from delivery artifact
#9 Change the taglib URI to match the fact that it is published via the API bundle

Fixed bugs

#8 NEBA core tag library may not be found during JSP compilation even though it is registered

Upgrading from 3.1.0

This release features a new URI for the neba tag library. To move to the new tag library, Update the taglib URI from

{% highlight java %} <%@taglib prefix="neba" uri="http://www.neba.io/core/1.0"%> {% endhighlight %}

to

{% highlight java %} <%@taglib prefix="neba" uri="http://neba.io/1.0"%> {% endhighlight %}

and subsequently delete the /var/classes folder in Sling to enforce re-compilation.

If you are using the delivery package, note that the classifier "aem" was removed from the artifact. The dependency is thus:

{% highlight xml %} io.neba io.neba.neba-delivery 3.2.0 zip {% endhighlight %}
