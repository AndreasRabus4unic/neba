Highlights of this release

On the road to NEBA 4

This release is the last planned 3.x release prior to NEBA 4, and thus the last planned release supporting legacy (<AEM 6.0) versions of Apache Sling and AEM. While issues of the 3.x branch will continue to be fixed, active development of new features will now be focusing on NEBA 4 (see NEBA 4 road map).

Improved fault tolerance

NEBA now prevents potential exceptions in Sling BackgroundHttpServletRequest(s) occurring when access to the session is attempted (#95) and tolerates null arguments to reference listeners (#102). Furthermore, the error message reported when an application attempts to synchronously access the resource resolver factory during a restart has been improved (#96).
