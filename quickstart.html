Step 3: Use NEBA

Write a resource model: Create a class anywhere inside the bundle.package.impl package namespace and annotate it with @ResourceModel. Cou may consult the @ResourceModel API docs or user guide on @ResourceModels to find out more. Feel free to use this snippet:

{% highlight java %} @ResourceModel(types = "cq:Page") public class PageModel { @Children private List children; public List getChildren() { return this.children; } } {% endhighlight %}

Use the model: Once your bundle is deployed, you can see your model in NEBA's model registry console in the Felix web console (/system/console/modelregistry).
Now, you can adapt any cq:Page resource to this model - for instance using a Spring controller:

{% highlight java %} @Controller public class MyController { @RequestMapping(value = "/page/childcount", produces = "text/plain") @ResponseBody public String getNumberOfChildren(@ResourceParam PageModel page) { return page.getChildren().size(); } } {% endhighlight %}

Once deployed, you can invoke this controller like so:

{% highlight bash %} GET /bin/mvc.do/page/childcount?page=/content/geometrixx/en {% endhighlight %}

More information on blueprint and Spring

Blueprint is part of the OSGi specification. NEBA relies on gemini-blueprint, the open source reference implementation of this standard.

A blueprint XML declaration is simply a Spring application context - with an additional blueprint namespace providing support for OSGi service exports and imports, SCR configuration creation and consumption and much more.
