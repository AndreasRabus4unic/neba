Installing NEBA

IDE

If you're using Eclipse, it is recommended to install the spring IDE plugin from this Update Site, and the M2eclipse plugin.

If you're using Intellij, we recommend using the Ultimate Edition, because of the Spring support that it provides.

Installing in Adobe AEM

Simply download the delivery zip (a CQ package) and install it in CQ via the CRX package manager. If you are upgrading from an older NEBA version uninstall and then delete the old NEBA II package in the CRX package manager, then upload and install the new package. A restart is not required.

Maven and NEBA

Make sure to integrate the (API) sources (e.g. mvn: eclipse:eclipse -DdownloadSources=true). The JavaDocs are also available in the repository. 

<dependency>
    <groupId>io.neba.delivery</groupId>
    <artifactId>delivery</artifactId>
    <version>1.0/version>
        <classifier>crx</classifier>
        <type>zip</type>
</dependency>

Bundle configuration

According to the OSGI blueprint spec, a blueprint XML must be present in /OSGI-INF/blueprint/. Take the following snippet and place it there. The snippet is a simple spring context (this is understood by blueprint by means of namespace handlers). Replace the "my.bundle.namespace" with your bundle's unique package namespace.

<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xmlns:osgi="http://www.springframework.org/schema/osgi"
       xmlns:context="http://www.springframework.org/schema/context"
       xmlns:aop="http://www.springframework.org/schema/aop"
       xmlns:bp="http://www.osgi.org/xmlns/blueprint/v1.0.0"

       xsi:schemaLocation="http://www.springframework.org/schema/beans
       http://www.springframework.org/schema/beans/spring-beans-3.0.xsd
       http://www.osgi.org/xmlns/blueprint/v1.0.0
       http://www.osgi.org/xmlns/blueprint/v1.0.0/blueprint.xsd
       http://www.springframework.org/schema/context
       http://www.springframework.org/schema/context/spring-context-3.0.xsd
       http://www.springframework.org/schema/aop
       http://www.springframework.org/schema/aop/spring-aop-3.0.xsd">

    <context:component-scan base-package="my.bundle.namespace" />
</beans>

And that's it, you're all set to write your first resource model.
